# AgentOS Launch Checklist

Use this before sharing AgentOS outside a private conversation.

## Repository

- [x] README explains the problem in the first screen.
- [x] Demo GIF is visible near the top.
- [x] Quick Start runs without model-token spend.
- [x] Documentation has a single index.
- [x] Contributing and security notes exist.
- [x] CI runs tests and smoke checks.
- [ ] Repository description is set on GitHub.
- [ ] GitHub topics are set: `ai-agents`, `sandbox`, `developer-tools`, `codex`, `mcp`, `ai-coding`, `code-review`, `automation`.
- [ ] Social preview image is configured.

## Demo

- [x] 60-second GIF exists at `docs/assets/agentos-demo.gif`.
- [ ] Terminal recording uses the latest CLI output.
- [ ] A tiny sample repo is available for repeatable demos.
- [ ] The demo shows original repo unchanged before approval.
- [ ] The demo shows sync preflight before sync.

## Product Clarity

- [x] Main wording avoids claiming production-grade security.
- [x] Main wording says AgentOS is not the AI brain.
- [x] Review-before-sync is the central message.
- [ ] Add a short real-world dogfooding note.
- [ ] Add a roadmap issue for a PR review bot / GitHub App path.

## Share Later

Do not launch as "an AI OS." Launch as a safety layer for AI coding agents.

Possible post titles:

- `AgentOS: run AI coding agents in a disposable workspace before syncing changes`
- `I built a review-before-sync layer for AI coding agents`
- `Let Codex work in a copied repo, then sync only approved changes`
